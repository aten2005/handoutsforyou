Classification of Physical Storage Media

  • Speed with which data can be accessed
  • Cost per unit of data
  • Reliability
  • Data Loss on Power failure or system crash
  • Physical Failure of the storage device

Volatile Storage

  • Loses content when power is switched off

Non Volatile Storage

  • Contents persist even when power is off

Cache

  • Fastest and mostly costly form of storage
  • Volatile

Main Memory

  • Fast Access, Generally too small
  • Too expensive to store the entire database
  • Volatile

Flash Memory

  • Data survives power failure
  • Reads are roughly as fast as main memory but writes are slow
  • Widely used in embedded devices such as cameras

Magnetic Disk

  • Data stored in spinning disk and read/written magnetically
  • Survives power failures
  • Primary medium for long term memory storage
  • Typically stores the entire database

Optical Storage

  • CD ROM = 640MB
  • DVD = 4.7-17GB
  • Slower than magnetic disk

Tape Storage

  • Non volatile, permanent
  • Mostly for archival and backups
  • Sequential access
  • Much slower than disk but very high capacity

Introduction

  • Data is stored in terms of blocks, when we are editing any record, the full block pertaining to this record is got onto ram and then updated and sent back
  • We cannot update single tuple independently

Storage Hierarchy

  • Cache(Costly, fast) → Main memory → Flash memory → magnetic disk → optical disk → magnetic tapes(Cheap, slow)
  • As you go right, space increases and speed decreases and cost decreases
  • Primary storage: Fastest media but volatile (cache, main memory).
  • Secondary storage: next level in hierarchy, non-volatile, moderately fast access time, also called on-line storage E.g. flash memory, magnetic disks
  • Usually connected to your system
  • Tertiary storage: lowest level in hierarchy, non-volatile, slow access time also called off-line storage E.g. magnetic tape, optical storage
  • Offline meaning we can remove it whenever we want

Magnetic Hard Disk Mechanism

  • Each disk pack contains disks and each disk contains 2 surfaces which can be used
  • Disk also called platter and are connected to a spindle
  • Each surface has concentric circles = tracks
  • Each track is divided into sectors and each sector is split into blocks
  • Same track on all the disks will lead to a cylinder
  • One block can be split across multiple sectors and you can change block size on an operating system level
  • It is advantageous to store data on same track on multiple surfaces than multiple tracks on same surface
  • It is preferable to store data in a single cylinder rather than multiple cylinders

Read Write Head

  • Positioned very close to the platter surface
  • Reads or writes magnetically encoded information

Surface of platter divided into circular tracks

  • Over 50k-100k tracks per platter on typical hard disks
  • Each track is divided into sectors
  • Each sector is typically sized at 512 bytes - many KBs
  • Typical sectors per track : 500 to 1000 (on inner tracks) and 1000 to 2000 on outer tracks

Read and Write a sector

  • Disk arm swings to position head on right track
  • Platter spins continually, data is read or written as sector passes under head

Head disk assemblies

  • multiple disk platters on a single spindle (1 - 5 usually)
  • one head per platter, mounted on a common arm
  • Cylinder i consists of i’th track of all the disks

Disk Controller

  • Interfaces computer system and the disk drive hardware
  • Accepts high level commands to read or write a sector
  • Initiates actions such as moving a disk arm to right track and actually reading/writing data
  • Computes and attaches checksums to each sector to verify the data is read correctly
  • If data is corrected, the checksum won’t match

Disk Subsystem

  • Multiple disks connected to a computer system through a controller
  • In Storage Area Networks, a large network of disks are connected by a high speed network to a number of servers

Performance Measures of Disks

Access Time

  • The time it takes from when a read or write request is issue to when the data transfer begins
  • Consists of
    • Seek time - time it takes to reposition the arm over the correct track
    • Rotational Latency - time it takes for the sector to be accessed to appear under the head

Data Transfer Rate

  • Rate at which data can be retrieved from or stored to the disk

Mean time to failure

  • The average time the disk is expected to run continuously without any failure
  • Typically 3-5 years

Optimization of Disk Block Access

  • Block is a contiguous sequence of sectors from a single track
  • Data is transferred between disk and main memory in blocks
  • If block size is too small, we require a large number of blocks
  • Sizes range from 512b to several kb
  • Smaller blocks → more transfers from disk
  • Larger blocks → more space wasted due to partially filled blocks
  • Typical block sizes today range from 4-16kb
  • Unused space of one block cannot be given to another block

Disk Arm Scheduling

  • Order pending accesses to tracks so that disk arm movement is minimized
  • Elevator Algorithm - Operating Systems Course will go deep into this

File Organization

  • Optimize block access time by organizing the blocks to correspond to how data will be accessed
  • Eg: Store related information on the same or nearby cylinders
  • Files may get fragmented over time
  • If data is inserted or deleted from the file
  • Or free blocks on disk are scattered and newly created file has it’s blocks scattered over the disk
  • Sequential access to a fragmented files results in increase in disk arm movement, hence we have defragmentation

RAID - Redundant Array of Independent Disk Systems

  • Disk organization techniques that manage a large numbers of disks providing a view of a single disk of
    • high capacity and high speed by using multiple disks in parallel
    • high reliability by storing data redundantly so that data can be recovered even if a disk fails
  • Different RAID levels have differing cost performance and reliability characteristics
  • Level 0 → Non redundant striping - same data is stored in multiple places but no redundancy
  • Level 1 → Mirrored Disks - Striping(High Speed)
  • Like this there are Level 0 to Level 6

Files and Records

  • A file is a sequence of records, where each record is a collection of data values (or data items)
  • Records are stored on disk blocks
  • The blocking factor for a file is the average number of file records in a disk block
  • A file can have fixed/variable length records
  • Unspanned → A single record only one block
  • For unspanned, we have 1024 bytes as block size and 100 as record size, then we have floor(1024/10) = 10 as blocking factor → 10 records per block
  • No of blocks will be given by ceil(size/blocking factor)

Record Organization

  • Can be spanned and unspanned while in file organization we have unordered files(heap) and ordered files(sequential)
  • A mapping will tell how many blocks have and where they are stored
  • Ordered files, searching is easy but in unordered files, insertion maybe easier

Multiple Clustering File Organization

  • Store several relations in one file using multi-table clustering file organization
  • Both tables get stored in the same file
  • DBA will take a decision regarding this
  • Good for queries which include both tables but bad for queries involving only one table
  • Results in variable size records

Data Dictionary Storage

  • The data dictionary also called system catalog stores metadata, that is data about data such as
    • Information about relations such as names, types, lengths of attributes
    • Users and accounting information including passwords
    • Statistical and descriptive data
    • Physical file organization information
    • Information about indices

Storage Access

  • A database file is partitioned into fixed length → blocks
  • Buffer → portion of main memory available to store copies of disk blocks
  • Buffer Manager → subsystem responsible for allocating buffer space in main memory

Example 1

  • Assume a disk pack which has 5 disks and data can be stored on both surfaces
  • We have 10 surfaces in total. All surfaces have uniform structure configuration
  • No of tracks per surface is 120. No of sectors per track is also uniform and sector size is 1024 bytes. Each track has 100 sectors
  • Compute size of each surface and of each cylinder and also compute total capacity of entire disk pack
  • Track = No of Sectors * Capacity of Sector = 100 * 1024 = 100kb = 0.1mb
  • Surface = = No of Tracks * Capacity of Track = 120 * 0.1mb= 12mb
  • Cylinder = No of surfaces * Capacity of Track = 5 * 2 * 0.1mb = 1mb
  • Disk Pack = No of Cylinders * Capacity of each cylinder = 120 * 1mb = 120mb

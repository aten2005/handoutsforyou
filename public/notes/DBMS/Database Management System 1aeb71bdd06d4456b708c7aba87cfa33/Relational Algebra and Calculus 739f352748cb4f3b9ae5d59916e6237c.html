Relational Algebra

  • Formal Query Language for relational data model
  • A basic set of relational model operations constitute the relational algebra
  • These operations enable the user to specify basic data retrieval requests
  • The result of a relational algebra query is also a new relation which may have been formed from one or more relations
  • A sequence of relational algebraic operations forms a relational algebraic expression, whose result is also a relation

Select

  • Used to select subset of tuples that satisfy some given condition
  • Denoted by σ\sigmaσ﻿
  • Example σdno=4\sigma _{dno=4}σdno=4​﻿ (EMP) selects all tuples from EMP table, where the value of the column dno is 4
  • General Format σcondition\sigma _{condition}σcondition​﻿ (EMP)

Projection Operation

  • Select’s certain columns
  • Symbol is π\piπ﻿
  • πname,age,dno\pi_{name, age, dno}πname,age,dno​﻿(EMP) selects columns name, age and dno

  • T ← R ÷\div÷﻿S
  • Selects all values for B column in R which contains all values under A in S
  • Hence the no column in T is only B

  • Natural join symbol is *
  • Used to join tuples from different tables based on some condition
  • Result is a new tuple with different arity

  • If we have 2 columns same between 2 tables and we take natural join, we get only those common columns ka tuples

Joins

Inner Join

  • Only combines tuples from R and S if they meet the given conditions
  • Tuples that do not meet the conditions are not shown
  • Usual type of join

Outer Join

  • Displays tuples of one of the relations even if there is no match for the tuple in other relation

Left/Right Outer Join

  • In the result relations, in addition to all the matching tuples from R and S, all the remaining tuples from left side also show up
  • For these tuples from say R on the left and S on the right, for R, columns under S will have null

Full outer join

  • All tuples from R and S appear with null values for other relation attributes

Group by

  • denoted by £
  • In exam use Script F and not pound symbol
  • Grouped by the value of some attribute and then aggregate functions are applied on individual groups
  • Dno £ count(ssn) (EMP) groups the tuples based on Dno and then applies count function on individual groups this will output the no of employees in each department
  • If SQL query is given as → Select dno, dname, avg(sal) from emp, dept where dno = dnum group by dno, dname;
  • In relational algebra duplicate tuples are removed by default

Tuple Relational Calculus

  • Formal query language for relational model
  • Relational calculus is a formal query language whereas relational algebra is procedural
  • Expression power for relational algebra and relational calculus is same
  • TRC is based on specifying a number of tuple variables
  • Each tuple variable ranges over a particular database relation
  • {t | condition (t)}
  • t → tuple variable and condition is a conditional expression involving t
  • Result is a set of all tuples that satisfy this condition
  • Example: { t | EMP(t) and t.salary > 50000}
  • Relation in this case is known as range relation
  • The above query retrieves all the attributes of relation EMP
  • Universal and existential quantifiers can be applied to tuples and if we use quantifiers then that variable is called a bound variable
  • { t.name, t.age | EMP(t) and (there is d) (Dept (d) and d.dname = ‘Research’ and d.dno = t.dno ) }

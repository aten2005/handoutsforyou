Introduction

  • Structured Query Language → most widely used commercial query language
  • High level declarative language to specify data retrieval requests for data stored in relational databases
  • Declarative because we tell it what to do and not how to do
  • SQL is relationally complete → everything done in relational algebra and calculus can be done in SQL
  • Supports views

Features of SQL

  1. DDL(Data Definition Language)→ Set of commands to support creation, deletion and modification of table structures and views
  1. DML(Data Manipulation Language) → Set of commands to pose queries, insert new tuples and update or delete existing tuples
  1. Embedded SQL → Allows users to call SQL from a lot of languages like C and C++
  1. Triggers → Actions executed by the DBMS whenever changes to the database meet specified conditions. Action to be performed and the set of conditions can be defined in “Triggers”.
  1. Transaction management
  1. Indexes can be created to speed up the access to data stored in DB

DDL Commands

CREATE

  • Create Table - CREATE TABLE DEPT(DNUM INT, DNAME VARCHAR(5), DLOC VARCHAR(10), PRIMARY KEY (DNUM), UNIQUE (DNAME));
  • Create Table with foreign key - CREATE TABLE EMP( EID INT PRIMARY KEY, ENAME VARCHAR(20), ESAL INT, DNO INT, ECITY VC(10), FOREIGN KEY (DNO) REFERENCES DEPT(DNUM));
  • CREATE TABLE PROJ( PNUM INT PRIMARY KEY, PNAME VARCHAR(10));
  • Create Table with 2 Primary key and 2 foreign key - CREATE TABLE EMP_PROJ( EINO INT , PNO INT, HRS INT, PRIMARY KEY (ENO, PNO), FOREIGN KEY (ENO) REFERENCES EMP(EID), FOREIGN KEY (PNO) REFERENCES PROJ(PNUM));

DROP

  • DROP TABLE EMP;

ALTER

  • Add Column ALTER TABLE EMP ADD ‘CITY’ VARCHAR(20);
  • Drop Column ALTER TABLE EMP DROP AGE CASCADE/RESTRICT;
  • Add Primary Key ALTER TABLE Persons ADD CONSTRAINT PK_Person PRIMARY KEY (ID,LastName);
  • Add Foreign Key ALTER TABLE Orders ADD CONSTRAINT FK_PersonOrder FOREIGN KEY (Person ID) REFERENCES Persons(PersonID);

CHECK

  • Used to put a constraint on the given data
  • CHECK (Age>=18)
  • To add it to an already existing table: ALTER TABLE students ADD CHECK (age >= 18);

DML Commands

  • Selecting tuples, columns
  • Inserting new tuples and updating existing tuples
  • Deleting existing tuples

SELECT

  • SELECT is used for retrieving information
  • Has no relationship with SELECT(σ\sigmaσ﻿) in relational algebra
  • SELECT attribute_list FROM table_list WHERE condition;
  • We can join tables directly using tables
  • SELECT * FROM qrcodes, studata WHERE studata.uid = "f20210075"; Gives data from both the tables as in it doesn’t combine any 2 columns
  • SELECT DISTINCT esal FROM emp; This command only gives distinct values compared to normal SELECT
  • We have unions, except and intersect in SQL

Substring Comparisons in SQL

  • % is used for arbitrary number of characters while _ is used for single character

BETWEEN

SELECT * FROM EMP WHERE (esal BETWEEN 30000 AND 40000) and dno= 5;

ORDER BY

  • Default ordering is ascending i.e. ASC
  • We specify DESC if we need descending order
SELECT * FROM Customers
ORDER BY Country ASC, CustomerName DESC;
// For multiple columns use this

NESTED QUERIES and EXISTS

SELECT E.eid, E.ename 
FROM EMP AS E 
WHERE E.dno IN(SELECT DNUM FROM DEPT WHERE DLOC = “chennai”);`
  • Here we have one query inside another and there is substitution going on
  • We have a NOT EXISTS term as well
  • EXISTS only checks if any tuple exists
  • Generally EXISTS and NOT EXISTS have correlations and correlations basically means inner query refers to an attribute from the outer query
SELECT E.eid, E. esal
FROM EMP AS E
WHERE NOT EXISTS (SELECT * FROM EMP_PROJ AS EP WHERE 
E.EID = EP.ENO);

JOIN Operations

  • We have join, natural join, left outer join and right outer join
  • If foreign key = null these are not counted in the response
  • Left outer join on table A, B includes all results of A no matter what
  • Right outer join on table A, B includes all tuples of B no matter what
  • We can argue that nested select conditions are also there but the issue is nested select conditions give cross product while join gives the intersection
  • Full Outer Join is a combination of Left Outer Join and Right Outer Join. Gives all the rows of the database. If for any data in the left table, there’s no row in the right table, then it gives only the left data with NULL values for right data and vice versa

Aggregate Functions

  • COUNT, SUM, MAX, MIN, AVG

GROUP BY

SELECT dno, count (*)
FROM EMP
GROUP BY Dno
HAVING count(*)>5;
  • Here only those attributes in the group clause will be displayed

INSERT

  • INSERT INTO table_name VALUES(v1, v2, ...vn);

DELETE TUPLE

  • DELETE FROM <table name> WHERE <condition>;
  • If we don’t specify condition, all are deleted

UPDATE

  • UPDATE table_name SET column_name = value WHERE condition;

Views in SQL

  • Single table derived from other tables
  • These other tables are called base tables
  • A view does not necessarily exist in physical form, it can be considered a virtual table
  • The tuples in a view are actually stored in various tables
CREATE VIEW EMP_DETAILS AS 
SELECT eid, ename, dname FROM
EMP, DEPT WHERE dno = dnum
  • View definitions are stored in the database but not the data of each view
  • From the next time, the view can be seen as a table and queries can be posed on it
  • A view is always up to date
  • Updates are generally not possible in views
  • Meant for querying only
  • Some times it’s possible to store views also for a specific time and these are called materialized views and we can use materialized views for caching
  • Every time you execute a view, the SQL statement is run

SQL Built In Functions

chr(int n)

  • Returns character given ascii number

concat(char1, char2)

  • Concatenates 2 strings

instr(string, char)

  • Select instr(’hello’, ‘e’) from dual;
  • Here instr returns 2

length(n)

  • Returns length of string

lpad/rpad(char1, n, char2)

  • lpad(’s’, 4, ‘d’) gives ddds
  • rpad(’d’, 3, ‘s’) gives dss

ltrim/rtrim(string, char)

  • trims char from string on left or right side

replace(string, search string, replacement string)

  • searches string for ss and puts rs there

substr(string, position, substring length)

  • gives a substring starting from position of length substring length

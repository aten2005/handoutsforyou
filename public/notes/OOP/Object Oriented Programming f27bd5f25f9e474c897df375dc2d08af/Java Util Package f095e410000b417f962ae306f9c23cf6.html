Type Wrappers

  • Java uses primitive types such as int, float etc
  • These do not inherit the object class and are not part of the object hierarchy
  • WrapperClass will convert primitive data types to objects
  • Primitive types are passed by value, so if we need to change the arguments passed into a method, it’s better to use the Object Wrappers
  • Object also supports synchronization in multithreading
  • We have Double, Float, Long, Integer, Short, Byte, Character, Boolean

Character

  • Constructor is Character(char c)
  • Call the charValue() function to return the character

Boolean

  • Constructor can be Boolean(Boolean boolValue) or Boolean(String boolValue)
  • Second constructor will identify true or false in upper or lower case
  • Call the boolValue() function to return the boolean

Numberic

  • valueOf of the Integer class can also be used to obtain an Object
    • static Integer valueOf(int val)
    • static Integer valueOf(String valStr) throws NumberFormatException
  • All these wrappers override the toString() one

Autoboxing and Auto Unboxing

  • Autoboxing is the process by which a primitive type is automatically encapsulated into it’s equivalent type wrapper whenever needed
  • Autounboxing is the reverse process and is the process by which the value of a boxed object is automatically extracted from a type wrapper when its value is needed

  • It is aimed to meet the following goals

HashSet

  • Extends AbstractSet and implements the Set
  • In hashing the information content of the key is used to determine the unique value called the hashcode
  • This hashcode is used to get the index of the element
  • Transformation of key into hashcode is done automatically and we never see it’s working
    1. HashSet(): It constructs a default hash set
    1. HashSet(Collection<? Extends E> c) : It initializes the hash set by using the elements of c.
    1. HashSet(int capacity): It initializes the capacity of the hash set to capacity. The default capacity is 16
    1. HashSet(int capacity, float fillRatio): The fourth form initializes both the capacity and the fill ratio (also called the load capacity) of the hash set from its arguments.
  • The fillRatio must be between 0.0 and 1.0 and it determines how full the hash set can be before it is resized upward.
  • When the number of elements is greater than the capacity of the hash set multiplied by its fill ratio, the hash set is expanded. Note: For constructors that do not take a fill ratio, o.75 is used.

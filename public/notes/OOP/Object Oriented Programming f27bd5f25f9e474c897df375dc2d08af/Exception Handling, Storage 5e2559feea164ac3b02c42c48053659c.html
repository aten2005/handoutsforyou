Exception Handling

  • Syntax Errors: Rules of the language not been followed
  • Runtime Errors: An operation that is impossible to carry out
  • Logic Errors: When the code does something else
  • An exception is an abnormal condition that arises in a code sequence at run time
  • Errors are exceptions that are not caught during normal run time of a program
  • Exceptions should be handles so that we fix the error and prevents the program from automatically getting terminated
  • 5 Important Keywords
    • try → monitor for exceptions
    • catch → catch the exception and handle it
    • throw → manually throw an exception
    • throws → thrown out of a class method
    • finally → absolutely must be executed after a try block. Gets executed even if there is a return in the try
  • When we have multiple catches, we need to remember that subclasses must come before super class else will lead to unreachable code compiler error
  • The throw is used to throw exceptions
    • throw ThrowableInstance
    • This instance is an object of Throwable Class
  • throws keyword

class NotEnoughBalance extends Exception {

Unchecked Exception

By extending RuntimeException, you can create unchecked exception:

class NotEnoughBalance extends RuntimeException {
     // Implementation
}

Storage in Java

  • Stack → Memory is allocated for local variables
  • Heap
    • More Stable Storage
    • Memory allocated in the heap remains in existence for the duration of the program, therefore static variables and methods are on the heap

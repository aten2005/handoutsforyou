Object

  • Object is an entity that has state, behavior and identity
  • State of the object has all the properties of the object and the current values of these properties
  • Behavior is how an object acts and reacts in terms of it’s state changes and message passing
  • Operations
    • Modifier → Alters state
    • Selector → Accesses State
    • Iterator → Permits all parts of object to be accessed
    • Constructor → creates an object and used to initialize the object
    • Destructor → Frees the state of the object

Classes

  • A class is a template for an object
  • It is important to understand that new allocates memory for an object during run time.
  • The advantage of this approach is that your program can create as many or as few objects as it needs during the execution of your program.
  • However, since memory is finite, it is possible that new will not be able to allocate memory for an object because insufficient memory exists. If this happens, a run-time exception will occur
Box b1 = new Box();
Box b2 = b1;
  • Here b2 is not being given separate space. Rather b1 is being referenced
  • Although b1 and b2 both refer to the same object, they are not linked in any another way.
  • For example, a subsequent assignment to b1 will simply unhook b1 from the original object without affecting the object or affecting b2
  • Say we do b1 = null, then b2 will still point to the original b1

Constructor

  • A constructor initializes an object immediately upon creation
  • It has the same name as the class in which it resides and is syntactically similar to a method

Garbage Collection

  • Java takes a different approach; it handles deallocation for you automatically.
  • The technique that accomplishes this is called garbage collection.
  • It works like this: when no references to an object exist, it is assumed to be no longer needed, and the memory occupied by the object can be reclaimed. There is no need to destroy objects explicitly. Garbage collection only occurs sporadically (if at all) during the execution of your program
  • The situation changes dramatically when you pass an object to a method because objects are passed by what is called-by-reference. Remember that when you create a class type variable, you only create a reference to an object.
  • Thus, when you pass this reference to a method, the parameter that receives it will refer to the same object as that referred to by the argument. This effectively means that objects act as if they are passed to methods using call-by-reference.
  • Changes to the object inside the method do affect the object used as an argument

Static Variables and Methods

  • The default starts from 0
  • Static variables can only be used inside static methods
  • Cannot refer to this or super
  • static methods can be called from non-static methods, but non-static methods cannot be called from static methods
  • the static block gets executed only once when the class is loaded
  • gets called before the constructor
  • static functions can be overridden

Final, Finalize method and Garbage Collection

  • final can be used to create constants
  • final can be used to prevent overriding
  • final can be used to prevent inheritance
  • finalize is an object method that runs when the object is about to be garbage collected
  • In java, garbage collection is automatic, as soon as the references to an object cease to exist, the java JVM removes the object from memory
  • A daemon Thread in java is a low-priority thread that runs in the background to perform tasks like garbage collection, finalizing etc.
  • It has no role other than to serve user threads
  • Garbage Collector can be called by System.gc(), and this need to be called to run the finalize function
  • protected void finalize()
  • finalize can run only once on an object

Object Cloning

  • clone() method is a way to create exact copy of an object
  • The java.lang.Cloneable interface must be implemented by the class whose object clone we want to create
  • Otherwise, we get CloneNotSupportedException
// A Java program to demonstrate shallow copy using clone()
import java.util.ArrayList;

// An object reference of this class is contained by Test2
class Test {
	int x, y;
}

// Contains a reference of Test and implements clone with shallow copy.
class Test2 implements Cloneable {
	int a;
	int b;
	
	Test c = new Test();
	
	public Object clone() throws CloneNotSupportedException
	{
	    Test2 clonedTest = (Test2) super.clone();
		  //commented part tells how to deep copy
		  // clonedTest.c = new Test();
		  // clonedTest.c.x = c.x;
		  // clonedTest.c.y = c.y;
			return clonedTest;
	}
}

// Driver class
public class Main {
	public static void main(String args[]) throws CloneNotSupportedException
	{
		Test2 t2a = new Test2();
		
		t2a.a = 10;
		t2a.b = 20;
		t2a.c.x = 30;
		t2a.c.y = 40;

		Test2 t2b = (Test2) t2a.clone();   // Creating a copy of object t1 and passing it to t2
		
		t2b.a = 100;	// Change in primitive type of t2 will not be reflected in t1 field
		t2b.c.x = 300;  // Change in object type field will be reflected in both t2 and t1(shallow copy)
		
		System.out.println(t2a.a + " " + t2a.b + " " + t2a.c.x
						+ " " + t2a.c.y);
		System.out.println(t2b.a + " " + t2b.b + " " + t2b.c.x
						+ " " + t2b.c.y);
	}
}

Issues

  • Object.clone() is protected, so we have to provide our own clone() and indirectly call Object.clone() from it.
  • Object.clone() doesn't invoke any constructor so we don't have any control over object construction.
  • If you want to write a clone method in a child class then all of its superclasses should define the clone() method in them or inherit it from another parent class.
  • Otherwise, the super.clone() chain will fail.
  • Object.clone() supports only shallow copying but we will need to override it if we need deep cloning.
  • Can be dangerous and can produce unintended results.

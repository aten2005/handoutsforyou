Introduction

  • Ports, buses, device controllers connect to various devices
  • Device drivers encapsulate device drivers, hides differences among IO controllers from IO subsystem of kernel
  • Signals from IO devices interface with computer
  • Port connection point for device
  • Bus set of wires and a protocol → PCI bus and expansion bus
  • Controller - electronics that operate port, bus and device
    • sometimes integrated on a chip
    • sometimes separate circuit board plugging into the computer, contains processor, microcode and private memory

IO hardware

  • IO instructions control devices
  • Controllers usually have registers where device driver places processor commands, addresses and data to write or read data from registers after command execution
  • Data in register - read by processor to get ip
  • Data out register - written by processor to send op
  • Status Register - contains bits that can be read by processor, bits indicate states like completion of current command, availability of byte to be read from data in register or occurrence of device error
  • Control register - written by processor to start a command

Polling

  • It is a mechanism where the CPU is always sending a request in order to check if the device needs attention
  • Host reads busy bit from status register until 0 → busy waiting done until not busy
  • Host sets read or write bit in command register and if write bit is set, writes a data byte into data out register
  • Host sets command ready bit in command register
  • Controller sees command ready bit and sets busy bit
  • Controller sees write command in command register, reads byte from data out register and completes the IO
  • Controller clears busy bit, error bit, command ready bit when transfer done
  • Step 1 is busy waiting of polling
    • Reasonable if device is fast but inefficient if device is slow
    • CPU switches to other tasks

IO Interface

  • Device types
    • Character stream or block
    • Sequential or random access
    • Sync or Async
    • Sharable or dedicated
    • Speed of operation
    • read-write, read only or write only
  • Block and Character Devices
    • Block devices include disk drives
    • Character devices include keyboards and mice
  • Types of IO
    • Blocking
    • Non blocking - input from keyboard and mouse
    • Async → disk and n/w IO

Other Aspects

  • Error handling
    • OS can recover from disk read, device unavailable, transient write failures
    • Retry a read or write
    • Most return an error number or code when IO request fails
    • System error logs hold problem reports
  • IO Protection
    • User process may accidentally or purposefully attempt to disrupt normal operation via illegal IO instructions
    • All IO instructions must be defined to be privileged and IO must be performed via system calls

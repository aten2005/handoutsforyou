Introduction

  • In computing, mmap is a POSIX-compliant Unix system call that maps files or devices into
    memory. It is a method of memory-mapped file I/O. It implements demand paging because file
    contents are not immediately read from the disk and initially use no physical RAM at all. The
    actual reads from the disk are performed after a specific location is accessed, in a lazy manner.
    After the mapping is no longer needed, the pointers must be unmapped with munmap.
  • Using mmap on files can significantly reduce memory overhead for applications accessing the
    same file; they can share the memory area the file encompasses, instead of loading the file for
    each application that wants access to it. This means that mmap is sometimes used for
    Interprocess Communication (IPC).
  • The main difference between System V shared memory (shmem), and memory mapped I/O
    (mmap) is that System V shared memory is persistent: unless explicitly removed by a process, it
    is kept in memory and remains available until the system is shut down. mmap'd memory is not
    persistent between application executions (unless it is backed by a file).

Dynamic Allocation using mmap

#include <stdio.h>
#include <sys/mman.h>

int main()
{
    int N = 5;
    int *ptr = mmap(NULL, N * sizeof(int), PROT_READ | PROT_WRITE, MAP_PRIVATE, 0, 0);
    if (ptr == MAP_FAILED)
    {
        printf("Mapping Failed\n");
        return 1;
    }
    for (int i = 0; i < N; i++)
        ptr[i] = i * 10;

    printf("The elements of the array => ");
    for (int i = 0; i < N; i++)
        printf("[%d] ", ptr[i]);
    printf("\n");

    int err = munmap(ptr, 10 * sizeof(int));

    if (err != 0)
    {
        printf("UnMapping Failed\n");
        return 1;
    }
}

Store file content using mmap?

#define _GNU_SOURCE

#include <stdio.h>
#include <sys/mman.h>
#include <stdlib.h>

int main()
{
    FILE *fp = fopen("input.txt", "r");
    if (fp == NULL)
    {
        printf("File not found\n");
        return 1;
    }

    int numberOfElements = 0;
    fscanf(fp, "%d", &numberOfElements);
    printf("Number of elements: %d\n", numberOfElements);

    int *ptr = mmap(NULL, numberOfElements * sizeof(int), PROT_READ | PROT_WRITE, MAP_PRIVATE | MAP_ANONYMOUS, 0, 0);

    if (ptr == MAP_FAILED)
    {
        perror("Mapping Failed");
        exit(EXIT_FAILURE);
    }

    for (int i = 0; i < numberOfElements; i++)
    {
        fscanf(fp, "%d", ptr + i);
    }

    fclose(fp);

    printf("The elements of the array => ");
    for (int i = numberOfElements - 1; i >= 0; i--)
        printf("[%d] ", ptr[i]);
    printf("\n");

    int err = munmap(ptr, 10 * sizeof(int));
    if (err != 0)
    {
        printf("UnMapping Failed\n");
        return 1;
    }
}

Create a C program that uses memory mapping to facilitate communication between parent and
child processes. The program should initialize an array of integers in the mapped memory
segment, display the initial values, fork a child process, have the child manipulate the array
(e.g., multiply each element by 10), and then display the updated values in the parent process

#define _GNU_SOURCE

#include <stdio.h>
#include <sys/mman.h>
#include <stdlib.h>
#include <sys/wait.h>
#include <unistd.h>
#include <sys/types.h>

int main()
{
    FILE *fp = fopen("input.txt", "r");
    if (fp == NULL)
    {
        printf("File not found\n");
        return 1;
    }

    int numberOfElements = 0;
    fscanf(fp, "%d", &numberOfElements);
    printf("Number of elements: %d\n", numberOfElements);

    int *ptr = mmap(NULL, numberOfElements * sizeof(int), PROT_READ | PROT_WRITE, MAP_SHARED | MAP_ANONYMOUS, 0, 0);

    if (ptr == MAP_FAILED)
    {
        perror("Mapping Failed");
        exit(EXIT_FAILURE);
    }

    for (int i = 0; i < numberOfElements; i++)
    {
        fscanf(fp, "%d", ptr + i);
    }

    fclose(fp);

    printf("Initially => The elements of the array => ");
    for (int i = 0; i < numberOfElements; i++)
        printf("[%d] ", ptr[i]);
    printf("\n");

    pid_t pid = fork();
    if (pid == 0)
    {
        printf("\nChild => The elements of the array => ");
        for (int i = 0; i < numberOfElements; i++)
            ptr[i] = 3 * ptr[i];
        for (int i = 0; i < numberOfElements; i++)
            printf("[%d] ", ptr[i]);
        printf("\n");
    }
    else
    {
        wait(NULL);
        printf("\nParent => The elements of the array => ");
        for (int i = 0; i < numberOfElements; i++)
            printf("[%d] ", ptr[i]);
        printf("\n");
    }

    int err = munmap(ptr, 10 * sizeof(int));
    if (err != 0)
    {
        printf("UnMapping Failed\n");
        return 1;
    }
}

Inodes

Linux must allocate an index node (inode) for every file and directory in the filesystem. Inodes
do not store actual data. Instead, they store the metadata where you can find the storage blocks
of each file's data. When a file is created on a system, a file name and Inode number are
assigned to it. Generally, to access a file, a user uses the file name, but internally file name is
first mapped with the respective Inode number stored in a table.

What happens if the inodes are full?

If the inodes are full in a file system, it means there is no available space to create more
files or directories, even if there is free disk space. Users attempting to create new files may
encounter errors indicating that there are no more available inodes. To address this,
administrators may need to resize the inode table or reformat the file system with a larger inode
ratio

Commands and Tools in Ubuntu

df command is used to check disk free space and df -ih helps in showing the number of inodes in human readable format

du command is used to check disk usage and the -a flag is used to display space usage counts for all the files as well. The output for these commands can be very large and one can use grep if they are looking for something specific.

The stat command in Ubuntu provides a detailed overview of a specified file or filesystem
(directory). When you run stat followed by a file or directory name, it provides a
comprehensive set of metadata, including size, block allocation, I/O block size, type (such as
regular file or directory), inode number, device ID, permissions, owner and group details, and
timestamps indicating access, modification, and status change

The fdisk command in Ubuntu is a disk partitioning utility that allows users to view and
manage disk partitions on their system. It provides functionalities such as creating, deleting, and
resizing partitions on a hard disk. When executed with the -l option, fdisk displays details
such as the start and end sectors, size, and file system type of each partition on all disks

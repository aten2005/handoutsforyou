OS vs Kernel

  • Kernel is part of OS
  • OS contains kernel, device drivers, utilities, system library and user interface
  • Kernel deals with memory management, disk management, process management and task management

Bootstrap Program

  • Loaded at power up or reboot
  • Initial program and is stored in ROM or EEPROM
  • Firmware → comes pre installed
  • Initializes all aspects of system like CPU registers, device controllers, memory contents
  • Locates and loads OS kernel and starts execution

Device Controller (HW)

  • Bridge between hardware device and OS or application program
  • Maintains some local buffer storage and a set of special purpose registers
  • Moves data between devices that it controls and it’s local buffer storage
  • OS has a device driver for each device controller

Device Driver (SW)

  • Understands device controller and provides the rest of the OS with a uniform interface with the device
  • IO devices and CPU can execute concurrently. Each device controller has a local buffer. CPU moves data from/to main memory to/from local buffer. IO is from the device to local buffer of the controller. Device controller informs the CPU that it has finished it’s operation causing an interrupt
  • Basically, local buffer has fast access time than real device, i.e. loading and extracting data
    from local buffer is faster than the device, and since CPU is really fast, local buffer were introduced to maintain efficiency of CPU
  • Local buffer is the part of device controller

Interrupts

  • IVT → Interrupt Vector Table → containing addresses of all service routines
  • IVT controls transfer to Interrupt Service Routine and executes the ISR and goes back to the starting place

Types of Interrupts

  • Hardware Interrupts: Maskable and Non Maskable
  • Software Interrupts: Normal and Exception/Trap

Storage

Main Memory

  • Random Access Memory
  • Only media, CPU can access directly
  • Volatile
  • Instruction Execution
  • Any location can be accessed and that is why called random access

Secondary Storage

  • Large, non volatile storage capacity
  • Stores both programs and data
  • Hard disk, Magnetic Disks
    • Metal/glass platters covered with magnetic recording material
    • Logically Divided into tracks and sectors
    • Disk controller determines the interaction between device and computer
  • SSD
    • Faster than magnetic disks due to less moving components

IO Operations

  • Device Driver loads registers within device controller
  • Device controller examines registers contents to determine what action to take
  • Controller starts transfer of data from device to local buffer
  • Device controller informs the device driver via an interrupt that it has finished it’s operation
  • Device driver returns control to OS

Direct Memory Access

  • DC transfers an entire block of data directly to/from it’s own buffer to memory
  • No intervention by CPU
  • Only one interrupt is generated per block to tell the driver that the operation has been completed
  • No interrupt per byte is generated and so the CPU is available for other works

OS Operations

  • Dual Mode: allows OS to protect itself and protect users from one another
  • User mode and kernel/supervisor/system/privileged mode
  • Mode bit provided by hardware
    • Provides ability to distinguish when system is running user code or kernel code
    • Some instructions - privileged and only executed in kernel mode
    • System call → mode to kernel → return resets it to user

Flow of the Process

  • User process → calls system call → trap → mode bit 0 → execute system call → locates ISR using interrupt vector table and executes it → returns and makes mode bit 1
  • Boot is in kernel mode → Hardware starts up in kernel mode
  • OS, User applications are in user mode
  • When trap interrupt occurs, hardware switches from user mode to kernel mode
  • Examples of Privileged instructions
    • IO Control
    • Switching to kernel
    • Timer management → Important for measuring infinite processes and loops for processes to get killed
    • Interrupt management

System Calls

  • Interface to the services provided by the OS
  • Typically these are written in C or C++ or some high level language
  • System calls are not directly used since these will not make code portable
  • Mostly accessed by programs via high level API
  • API specifies a set of functions that people can access through itself
    • Win32 API
    • POSIX API → Portable OS Interface for UNIX
    • Java API → JVM
  • Each system call is a number and we have a system call interface which has an index for these numbers

Types of System Calls

Process Control

  • Create process
  • Terminate Process
  • End, Abort
  • Load, execute

File Management

  • Create, delete, open, close, read, write files
  • Get and set file attributes

Device Management

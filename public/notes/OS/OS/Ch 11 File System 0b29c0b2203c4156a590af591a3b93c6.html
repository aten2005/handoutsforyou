File Attributes

  • Name
  • Identifier - unique tag - number or token identifies file within file system, non human readable
  • Type
  • Location - pointer to file location on device
  • Size - current file size
  • Protection and permissions - controls who can do reading writing and executing
  • Time, data and user identification - data for protection, security and usage monitoring
  • Information about files are kept in directory structure maintained on the disk
  • Directory entry - name and identifier

File Operations

  • System calls for file operations
  • create, write, read, reposition within the file(seek), delete, truncate, append, rename
  • OS maintains an open file table
  • For a requested file operation, file is specified via an index into the table
  • When file is closed, OS removes entry from file table

File Types

Access Methods - Sequential and Direct (Random)

Directory

  • The directory is organized logically to obtain
    • Efficiency - locating file quickly
    • Naming - convenient to users
      • Two users can have same name for different files
      • The same file can have several different names
    • Grouping - logical grouping of files by properties
  • Directory operations
    • Search for a file
    • Create a file
    • Delete a file
    • List a directory
    • Rename a file
    • Traverse the file system

Single Level Directory

  • A single directory for all users
  • Entire system will contain only one directory which is supposed to mention all the files present in the file system
  • Directory contains one entry for each file present in the file system
  • Naming and Grouping Problem → There’s not a lot of logical grouping possible

Two Level Directory

  • Separate directory for each user
  • Username and file name define a path
  • MFD (Master File Directory) is indexed by username or account number → it will contain number of users entries
  • Can have the same file name for different user
  • Efficient searching, only UFD (User File Directory) is searched for creation or deletion
  • Creation and deletion of user directories - admin
  • Sharing is not possible

Tree Structured Directory

  • Efficient searching, but Directory is a special file
  • Creating a new file is done in current directory
  • Deleting a file with rm command
  • Creating a new subdirectory is done in current directory
  • With permissions, one can access another’s files
  • Absolute Path Name → root/spell/mail/copy/all
  • Relative path name from mail /copy/all

Acyclic Graph Directory

  • Have shared subdirectories and files
  • Two different names
  • Can result in Dangling Pointers
  • We have two different absolute paths for one file

General Graph Directory

  • How do we guarantee no cycles?
    • Allow only links to file not subdirectories
    • Garbage Collection - mechanism to determine when the last of the references of a file has been deleted and disk space can be reallocated
    • Every time a new link is added, use a cycle detection algorithm to determine whether it is ok

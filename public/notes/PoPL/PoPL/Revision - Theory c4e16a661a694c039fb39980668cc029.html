12. Functional Languages

  • Made to mimic mathematical functions to as much extent as possible
  • There’s no concept of variables which implies that the programmer need not worry about allocating and deallocating space
  • There’s no iterations and every loop has to happen in the form of recursion
  • Based on referential transparency which says that the same function for the same input will always give the same output
  • This helps in testing cause we do not have to worry about the context of the function while testing
  • Haskell is pure functional
  • We have ML, OCaml, LISP also which has lists

11. Logic Programming

  • Logic programming deals with the use of symbolic logic and inferences from propositions. Prolog is one example of a logic programming language
  • In prolog, the propositions have no intrinsic semantics and a variable can be a term
  • Goals are known as queries and if you have multiple goals, then the smaller ones are called subgoals
  • Logic programming is used for defining propositions, defining relationships between propositions and inferencing answers for queries using these propositions
  • We have quantifiers as well which include, for all and there is
  • Headless horn clause → no LHS

10. Exception Handling

  • Exception handling is the process of finding unusual activity in the program, taking action against it and ensuring the program runs as usual
  • The code which processes all this is called the exception handler
  • Program Language implementors have written instructions in the language for implicit as well as explicit exceptions.
  • Implicit exceptions are raised by default while explicit exceptions or those defined by the user have to be triggered by the user
  • In java all exceptions are part of throwable class
  • We have checked as well as unchecked exceptions. Checked exceptions mostly get fixed after compilation but the unchecked ones occur at run time and the user has to deal with these
  • Ada has a lot of exception handling mechanism
  • C++ has no inbuilt exception handling other than the one in the standard library
  • Whenever an exception occurs, the code can either decide to terminate or resume and continue
  • Java recently introduced the assert keyword. Using the assert key word the programmers can put built in checks in to the code. For example, say before dividing you want to check if the denominator is greater than 0 or not you can use the assertion there and if the assertion fails, an ASSERTIONERROR is thrown
  • Event is a notification that something has occurred, even handling is done using event listeners in java and these event listeners need to be told what event to look out for x

9. Concurrency

  • Concurrency is the process of running multiple processes or applications at the same time parallelly. There’s two types of concurrency, physical and logical. Physical concurrency is when there are multiple processors and each processor runs one process while logical concurrency is where you have only one process which is context switching between both the processes to give you the feel of concurrency
  • Heavy weight tasks have their own address space while light weight tasks run in the same address space
  • Light Weight tasks have 3 advantages
    • Easy to implement
    • Easy to monitor their execution
    • IPC between these is easy through non local variables and if there is no tIPC then they are disjoint
  • Motivation behind concurrency
    • Running the same program on multiple processors concurrently can produce faster output
    • Even if there is only one processor, we can use threads to run the program concurrently which might be faster than non concurrent
    • Multiprocessing helps as a new way of thinking for some problems similar to how recursion exists
    • In distributing computing, we use multiprocessing to scale the processes to handle a lot of requests
  • Synchronization
    • The concept of handling multiple concurrent request without leading to loss in data integrity
    • We have two types of synchronization
    • Cooperative → Where A is waiting for B to complete and then only will get to proceed
    • Competitive → When A and B are trying to access the same resource but the resource can only be accessed by one process at one time
  • Methods for Synchronization
    • Monitors → Monitor is an encapsulation mechanism through which only the synchronized methods are available to the user such that only one process can access the object inside and modify it at one point of time
    • Semaphores → It is a data structure that holds an integer and a queue of tasks which keeps track of when a task can be performed. Semaphores can be used for both coop and comp sync
    • Message Passing → It is used to send messages between processes to keep track of them and synchronize them

7. Subprograms

  • Subprograms are part of process abstraction where each subprogram
    • has one entry point
    • caller is suspended when the subprogram is in suspension
    • control always returns to the caller once the execution of the called ends
  • Parameters
    • Formal parameters - dummy parameters that only get binded once the subprogram is executed
    • Actual parameters - the list of the parameters bound to the formal ones
    • Positional parameters - When the binding between AP and FP has to happen in order of their passing, we call it positional parameters
    • Keyword parameters - used when lists are wrong and it becomes hard to keep track of positions, python, perl support keyword
  • Procedures don’t return anything while functions always return a value
  • Pass by value - IN
  • Pass by result - OUT
  • Pass by value-result - INOUT
  • Pass by reference - INOUT → address is passed
  • Pass by name - INOUT → we directly substitute name wise

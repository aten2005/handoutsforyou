What arguments can you make for the idea of using a single programming language for all domains?

  • If answer is yes, the arguments would be you dont have to learn different languages
  • Finding people with that language knowledge will be easier than when there are a lot of languages
  • Easier access to developer tools since we have only one language
  • Maintenance also becomes easier because there's only one language and it becomes universal

What arguments can you make against the idea of using a single programming language for all domains?

  • Programming language will be too large and complex
  • Compilers will be complex and costly and may not generate efficient code
  • May not be suitable for all domains
  • Instead of concentrating on what needs to be done you'll be looking at figuring out how the implementation and design is working on the language

What fragment ruins readability of a language?

  • GOTO
  • Switch
  • Modular or monolithic → prefer modular
  • Function calls
  • Break
  • Continue
  • Can cause infinite loops
  • Readability and maintainability is affected
  • Difficult to read and understand

Java uses right braces to end all compound statements, what are arguments that are for and against the design?

  • Pros are that we know where it starts and where it ends and improves simplicity
  • Cons is that we don't know which statement ends where

Give pros and cons of multi line comments and single line comments

  • Multi line comments can have better descriptions
  • Multi line one disadvantage is that we may forget the pair
  • Potential lines of code may go inside the comments
  • Diminished reliability
  • Think of it from an interpreted language pov
  • It stops giving any output if we forget the comment closing
  • Disadvantages of single line comments is you have to write them multiple times

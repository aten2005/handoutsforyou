Attribute Grammar

  1. Syntax Rule: <assign> → <var> = <expr>

    Semantic Rule: <expr>.actual_type == <var>.actual_type

  1. Syntax Rule: <expr> → <var>[2] + <var>[3]

    Semantic Rule: <expr>.actual_type ← if (<var>[2].actual_type = float) and (<var>[3].actual_type = float) then float else if (<var>[2].actual_type = int) and (<var>[3].actual_type = int) then int else mixed end if

    Predicate: <var>[2].actualtype == <var>[3].actualtype

  1. Syntax Rule: <expr> -> <var>

    Semantic Rule: <expr>.actual_type <- <var>.actual_type

  1. Syntax Rule: <var> → A | B | C

    Semantic Rule: <var>.actual_type ← look-up(<var>.string)

Write an attribute grammar that would facilitate computing the value of expression: expr → id + expr | id * expr | (expr) | id and id → A | B | C

  • We have 5 rules here
  • Syntax Rule: expr → id + expr[1]
    • Semantic Rule: expr.val ← id.val + expr[1].val
  • Syntax Rule: expr → id * expr[1]
    • Semantic Rule: expr.val ← id.val *expr[1].val
  • Syntax Rule: expr → (expr[1])
    • Semantic Rule: expr.val ← expr[1].val
  • Syntax Rule: expr → id
    • Semantic Rule: expr.val ← id.val
  • Syntax Rule: id → A | B | C
    • Semantic Rule: id.val ←lookup(id.string)

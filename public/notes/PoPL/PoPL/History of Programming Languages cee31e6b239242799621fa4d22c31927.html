Plankankul

Pseudocode

  • Every code was written in machine code
  • To make an addition of instruction, all instructions that refer to addresses that follow the addition must be found and modified
  • For deletion → replace with no operation → nop

Short Code

  • Was not translated to machine code
  • Implemented using pure interpreter
  • It was called automatic programming
  • 50 times slower than machine code

Speed coding

  • Add instruction took 4.2ms
  • After loading interpreter only 700 words spaca=³e left

IBM 704 and Fortran

  • All floating operations were simulated before
  • Included indexing and floating point instructions in hardware(way faster than simulating in software)
  • Formula Translation
  • FP
  • First compiled high level language
  • Environment in which fortran was developed
    • Small memory, slow, unreliable
    • Primary use was for scientific computations
    • No existing effective and efficient programming languages
    • Cost of HW was high
  • Fortran 1 has variable names upto 6 characters while it was just 2 in fortran 0
  • Variable names that began with I, J, K, L, M, N were implicitly integer type and all others were implicitly floating point
  • FORTRAN 6 → ANSI 66

Fortran 2

  • Added independent compilation of subroutines
  • Longer programs had a poor chance of being compiled completely before a machine failure occurred in fortran 1

Later Fortrans

  • Fortran 3 was lite
  • Fortran 4 very widely used
  • Fortran 77 came next
    • Character string handling
    • else also with if
  • Fortran 90
    • Had recursion also
    • Dynamic Arrays
    • Pointers
    • Records
    • Multiple Selection Statements
    • Modules
      • FORTRAN became fortran
  • Fortran 2003 → OOP

Effect of FORTRAN

  • Type and storage of all variables is fixed before runtime. Hence allows optimizing compilers.
  • No new variables or storage allocation at runtime, hence more efficient and less flexible
  • No recursion
  • They start with program and end with end program followed by program name
  • We should always use implicit none that allows the compiler to check that all your variables types are declared properly
  • Case Insensitive

Functional Programming and LISP

  • List Processing → required some methods to allow computers to process symbolic data in linked lists. At that time most computations were on data in arrays
  • 1950s → It was based on applying functions to arguments
  • Need came because of some methods to allow computers to process symbolic data in linked lists
  • Requirement → Recursion, Conditional expressions, Dynamic Allocation and Deallocation of linked list space
  • Though fortran didn’t support these, FLFP → Fortran list processing language → did not have recursion
  • Functional Programming → bind everything in pure mathematical functions style
  • Declarative type → you only focus on what should be done → Example: SQL
  • Variables are immutable
  • Loops of for and while were implemented through recursion
  • Pure LISP → applying functions to arguments, has atoms and lists
  • Each node has two pointers and represent a list element
  • A node containing an atom has it’s first pointer pointing to some representation of that atom such as it’s symbol, numeric values or a pointer to a sublist
  • Descendants of LISP
    • Scheme → treat functions as first class entities
    • Common LISP → relatively large and complex
    • Other examples: haskell, caml, miranda, meta language

Advantages

  • Easy to read, understand
  • parallelism since a function can’t change anything outside

Disadvantages

  • Recursion can affect readability, performance
  • combining and IO is difficult
  • Recursion is more costly because in the layers of program execution → code → heap → stack

ALGOL 58

  • Algorithmic Language
  • Descendant of fortran
  • To introduce PL that is not tied to hardware, more flexible and powerful i.e. combination of simplicity and elegance
  • Reason for failure → IBM and it’s scientific user group SHARE abandoned ALGOL 58

ALGOL 60

  • ALGOL 60 came into existence and used BNF → Backus Naur Form
  • Concept of block structure, 2 different types of parameter passing
  • Declare space for arrays in the frame
  • Descendants → PLI, C, PASCAL, Java, C#
  • Machine Independent
  • Too flexible → difficult to read and understand
  • Lack of input, output statements

COBOL

  • Common Business Orientated Language
  • Widely used and compiled language
  • Progenitor is FLOW MATIC → ancestor
  • Data processing programs must use english like words rather than math expressions
  • Preprocessor → define → first used in cobol
  • First to use macros
  • Hierarchical Data structures were introduced
  • Data division → 4 different divisions
    • Identification → name of program
    • Environment → from where you are taking input and output sources
    • Data → declare all your variables
    • Procedural → statements that are executed
  • Mandated by DOD
  • Led to Electronic mechanization of accounting

BASIC 1971

  • Very little respect
  • Ignored by Computer Scientists
  • Had a meager set of control statements
  • Very popular in microcomputers
  • Strong resurgence appeared in 1991 when visual basic (.NET)
  • Easy for non CS students to learn and fast turnaround for homeworks
  • Free and Private Access
  • Access and Activity of each user was private
  • Early versions very small and non interactive → All data should be input before execution starts
  • First widely used language through terminals
  • GUI applications

PL/I Programming Language Interface

  • First large scale attempt to design a language that could be used for broad spectrum of application areas
  • Recursion + block structure → lisp
  • Report generation from cobol
  • Communication through global data from fortran
  • Allow concurrent execution of subprograms
  • 23 different types of exceptions
  • Pointer data types were introduced
  • Any construct which was useful came onto PLI → language became very large → misunderstanding and complex

Dynamic Languages

APL

  • Started allowing dynamic typing
  • Dynamic storage allocation
  • A programming language
  • Large number of operators and symbols
  • Highly expressive

SNOBOL

  • String pattern matching
  • Text Processing

SIMULA 67

  • Extension of ALGOL 60
  • Concept of coroutines → all methods have similar status
  • Restore execution of subprogram where it was left
  • First → Supported abstraction
  • Supported class in a limited way
  • Led to object oriented paradigm

ALGOL 68

  • Orthogonality
  • Less constructs → multiple operations
  • Descendants → Pascal and C
  • User defined types are significant because they allowed users to design data abstraction that suits their problem solution

PASCAL

  • Simple, expressive and hence popular
  • It was impossible to write subprograms which take array as argument
  • Designed for teaching

C Language

  • CPL(1960) → BCPL(1967) → B(1970) → NB → C
  • ANSI and ISO standardized C
  • Adequate control statements and data structures
  • Suits many application areas
  • Rich set of operators provide good expressiveness
  • It’s compiler was part of Unix in 1980 hence gained popularity

Logic Programming

  • Use of formal logic notation to communicate computational process to a computer
  • Predicate calculus is the notion used in current logic programming languages
  • These are non procedural
  • We supply:
    • Relevant Information (facts)
    • Inferencing process (rules)

Prolog → Programming in Logic

  • Fundamental design of prolog was done in 1970
  • Prolog interpreter developed in 1972
  • Common use of a prolog is a kind of intelligent database
  • Contains two kinds of statements: facts and rules
  • Fact Statements
    • mother(joanne, jake)
    • mother(vern, joanne)
    • mother(x, y)
    • father(m, n)
  • Rule statements
    • grandparent(a, c) :- parent(a, b), parent(b, c)
  • Prolog database can be interactively queried with goal statement
  • When such a query/goal is presented to the prolog system, it uses it’s resolution process to attempt to determine the truth of the statement
  • If it can conclude the goal is true, it displays true else it displays false

Ada

  • Developed for DoD in 1975
  • Packages of Ada provide encapsulation → data objects + procedures
  • This in turn supports abstraction
  • Supports exception handling
  • Generic units can be written. This increases reusability
  • Allows concurrent execution of special program units called tasks
  • Widely used in commercial. Defense, avionics etc.

Small talk

  • First full fledged OOPL
  • Small talk world is populated by nothing but objects, integer constants to complex software systems
  • All computations happen by sending messages
  • We have object abstraction as class similar to Simula67
  • OO ideas came from simula but matured with small talk

C++

  • It is C with classes and is hybrid (procedural and OO)
  • It supports both functions and methods
  • Supports both procedural and OO programming
  • Operator overloading
  • Has pointers. Dynamic binding supported
  • Supports multiple inheritance
  • Classes and methods can be templated
  • Supports multiple inheritance
  • Supports exception handling

Objective C

  • Dynamic binding of messages to objects, included garbage collection

Delphi

  • Object oriented features added to Pascal and this was more elegant and safer than C/C++
  • No operator overloading

Go

  • Borrows syntax from C,
  • Faster compilation although does not support traditional OO as there is no form of inheritance
  • We can define methods for any type
  • Supports pointers, goto, interfaces and concurrency

Java

  • Java → Sun microsystems
  • Embedded consumer electronics
  • Became popular with WWW

Scripting Languages

  • Earlier used for putting list commands called scripts in a file to be interpreted
  • The first of this is called sh(shell) began as a collection of commands that were interpreted as calls to system subprograms that performed utility functions such as file management and simple filtering
  • To this were added variables, control flows, functions and other capabilities
  • Examples: ksh, awk etc.
  • They are interpreted
  • Perl → combination of sh and awk
  • Java Script → related to java and dynamically creating html docs
  • PHP: Hypertext Preprocessor → It is a server side scripting language. Code embedded in HTML es executed at server when the HTML is requested by the client. Similar java script syntax
    • Used to write programs that need web access to databases
  • Python → OO interpreted scripted language.
  • Ruby
  • C#

Markup Languages

  • XML → Extensible Markup Language → used to define data docs
  • XSLT → Extensible Stylesheet Transformation Language
  • JSP → Java Server Pages

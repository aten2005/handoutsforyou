LL Parser

  • You start with S in stack and keep pushing the right hand side and end with empty

Example 1

- S→ aM and M → bMa | a and we have abbaaa
- Stack is represented by BOTTOM —> TOP
- LMD: S → aM → abMa → abbMaa → abbaaa

- We use stack → push down stack
- Stack: S
- Stack: M a
- Stack: M (pop a cause matches with input’s first character)
- Stack: a M b
- Stack: a M (pop b)
- Stack: a M
- Stack: a a M b
- Stack: a a M (pop b)
- Stack: a a a
- Stack: (pop a) x 3
- Stack: EMPTY!!

Example 2

- S → aAc and A → aA and A → b and we need aabc

- Stack is represented by BOTTOM —> TOP
- Stack: S Input: aabc
- Stack: c A a Input: aabc
- Stack: c A Input: abc
- Stack: c A a Input: abc
- Stack: c A Input: bc
- Stack: c b Input: bc
- Stack: empty Input: empty

Shift Reduce or LR Parser

  • You start with empty on stack and keep adding characters and end with
  • Note: Correct RHS is called the handle → It’s a substring in the sentential form
  • Also called as shift reduce algorithms. A parser is a PDA
  • The input is examined from left to right, one symbol at a time.
- S → aAc and A → aA | b

- S ⇒ aAc → aaAc → aabc

- Stack: Empty (Shift a)
- Stack: a → (Shift a)
- Stack: a → a → (Shift b)
- Stack: a → a → b → (Reduce b to A using A → b)
- Stack: a → a → A → (Reduce aA to A using A → aA)
- Stack: a → A → (Shift c)
- Stack: a → A → c (Reduce aAc to S using S → aAc)
- Stack: S

Attribute Grammar

  1. Syntax Rule: <assign> → <var> = <expr>

    Semantic Rule: <expr>.expected_type ← <var>.actual_type

  1. Syntax Rule: <expr> → <var>[2] + <var>[3]

    Semantic Rule: <expr>.actual_type ← if (<var>[2].actual_type = int) and (<var>[3].actual_type = int) then int else real end if

    Predicate: <expr>.actual_type == <expr>.expected_type

  1. Syntax Rule: <expr> -> <var>

    Semantic Rule: <expr>.actual_type <- <var>.actual_type

    Predicate: <expr>.actual_type <- <var>.actual_type>

  1. Syntax Rule: <var> → A | B | C

    Semantic Rule: <var>.actual_type ← look-up(<var>.string)

The look up function looks up the variable name in the symbol table and returns the variable’s type
